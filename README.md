# STAF Antenna Layout Lab

スタッフ株式会社の板金アンテナを顧客（無線設計者）が自社基板条件で検討するためのWebプロトタイプです。
基板サイズ・外形・GND構成・アンテナ配置を変更すると、選択した周波数帯のVSWR概算と採用可否の目安が連動して表示されます。

## 対応機種

| 型番 | 種別 | 対応帯域 | 概算係数 |
|---|---|---|---|
| 1018-521A | SMT対応省スペース板金アンテナ | 2.4GHz帯 / 800-900MHz帯 / 1.7-2.7GHz帯 | 基準調整済 |
| 1019-107A | Wi-Fi 7 / BLE対応トライバンドSMD板金アンテナ | 2.4GHz / 5GHz / 6GHz帯 | 仮係数 |
| 1018-456A | 挿入実装対応板金アンテナ | 800-900MHz帯 / 1.5-2.1GHz帯 | 仮係数 |

寸法・周波数帯・実装方法は公式製品ページ（ https://www.staf.co.jp/product/antenna/ ）の公開仕様に基づきます。

## 起動

PowerShellでこのフォルダへ移動し、同梱のNode.jsサーバーを起動します。

```powershell
node server.mjs
```

ブラウザで `http://127.0.0.1:4173` を開きます。
ビルド工程はなく、静的ファイル（HTML/CSS/JS）のみで動作します。

## ファイル構成

### 公開用（HPへアップロードするのはこの4ファイル＋imagesフォルダ）

- `index.html` — 画面構造（機種選択 → 実装条件 → VSWR判定）
- `antenna-data.js` — **機種カタログ**。寸法・帯域・概算モデル係数をデータとして保持
- `app.js` — 選択中の機種＋帯域でUI全体を駆動するロジック
- `styles.css` — スタイル
- `images/staf-logo.svg` — ヘッダー・フッターの会社ロゴ（ベクター）
- `images/1018-521A.png` / `images/1019-107A.png` / `images/1018-456A.png` — 白背景を透過処理した製品画像
  （`images/*_raw.*` と `images/process.py` は透過処理の作業用ファイルのため、アップロード不要）

### 社内用（HPへはアップロードしない）

- `editor.html` / `editor.js` — **機種データ編集ページ**。公開ページからのリンクは無く、アップロードしなければHP上からは見えません・使えません
- `server.mjs` — ローカル確認用の簡易HTTPサーバー
- `README.md` — このファイル

## 機種を追加するには（推奨: 編集ページを使う）

ローカルで `editor.html` を開くと、フォーム入力だけで機種データを作成できます。

1. 「＋ 新規機種を追加」を押し、型番・製品名・寸法・画像URLなどカタログ値を入力
2. 帯域ごとに利用帯域の下端・上端 (MHz) を入力し、「表示範囲・確認周波数を自動入力」→「概算係数を自動算出」
   （基準条件で帯域端がVSWR 2.0以下に収まるかを自動チェックします）
3. 「カタログへ反映」→「antenna-data.js を生成」→ コピーまたはダウンロード
4. 生成された内容で `antenna-data.js` を置き換え、HPへアップロード

編集内容はブラウザ内（localStorage）に下書き保存されます。`antenna-data.js` を置き換えるまで公開ツールには影響しません。

## 機種を追加するには（手動で編集する場合）

`antenna-data.js` の `ANTENNA_CATALOG` 配列にエントリを1つ追加するだけで、
機種選択カード・周波数帯切替・VSWR概算・判定文言のすべてに反映されます。

各帯域に必要な係数:

- `startMHz / endMHz` — 利用帯域（判定・カバー率の範囲）
- `chartStartMHz / chartEndMHz` — グラフ表示範囲
- `checkMHz` — 帯域下端・中央・上端の確認周波数（3点）
- `f0Base / minBase / halfWidthBase` — 基準条件での共振点・最小VSWR・帯域幅係数
- `refBoard` — その帯域での推奨基準基板サイズ（「初期値へ」ボタンの値）
- `calibrated` — 実測に基づく校正済みかどうか（`false` の機種は画面に「仮係数」表示）

## スタッフHPへの掲載について

静的ファイルのみで構成しているため、次のいずれかで掲載できます。

1. **iframe埋め込み** — 公開用ファイル一式（4ファイル＋`images/`）を任意のパスに配置し、製品ページから `<iframe>` で参照
2. **直接ホスティング** — `index.html` / `styles.css` / `app.js` / `antenna-data.js` と `images/` フォルダ（透過済みPNG 3枚）をサイトにアップロード（サーバーサイド処理は不要）

**注意: `editor.html` と `editor.js` は社内編集用のため、HPへはアップロードしないでください。**
公開ファイルからのリンクは一切無いため、アップロードしない限り外部からは見えません。
機種を追加・修正したときは、編集ページで生成した `antenna-data.js` だけを差し替えれば反映されます。

製品画像は白背景を透過処理したローカルPNG（`images/`）を参照しています。ロゴは `images/staf-logo.svg`（ベクター）をローカル参照しています。

## 重要

VSWRはユーザー体験検証用の概算モデルであり、電磁界解析や製品性能保証ではありません。
1018-521Aの2.4GHz帯以外の係数は仮値です。実運用版では、実測値・電磁界解析結果・整合回路条件を基にしたサロゲートモデルへ置き換える想定です。

板金アンテナは基板端への実装が設計前提のため、アンテナを基板の内側（最寄り基板端から `EDGE_LIMIT_MM`＝既定5mm を超える位置）へ配置した場合は**設計対象外**とし、VSWR等を表示せずNG判定にします（`app.js` の `EDGE_LIMIT_MM` で調整可能）。
